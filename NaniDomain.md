# 도메인 로드맵 - 나는 어떤 전문가인가

작성 20분 · Module 01에서 1회 작성 → M08 해커톤 후, 수료 시 두 번 갱신합니다. 이 페이지는 120일 동안 여러분의 나침반입니다.

## 1. 내 도메인 한 줄 정의

직함이 아니라 '내가 해결하는 문제'로 적어보세요. 예: "환자가 진료 내용을 이해하고 실천하게 만드는 일" (X 내과 의사)

> 나는 고등학교의 교육과정과 학생의 학습환경 및 진로를 이해하고 서류를 통해 이를 평가하는 사람이다.

## 2. 현재 위치 자가진단 — 효율 × 혁신

솔직하게 1~5점. 두 점수가 여러분의 사분면 위치이고, 세션에서 어느 트랙 비중을 높일지의 근거가 됩니다.

| 축 | 질문 | 점수 (1~5) |
| --- | --- | --- |
| 효율 (루틴) | 내 분야의 익숙한 문제를 빠르고 정확하게 처리하는가 | 4 |
| 혁신 (적응) | 처음 보는 문제에 기존 절차를 벗어난 해법을 시도하는가 | 3 |

→ 효율이 낮다면: 초반 세션은 루틴 트랙(AI 코치와 기본기) 비중을 높게. 혁신이 낮다면: 문제 카드에 먼저 부딪히는 혁신 트랙부터.

## 3. 내 성장 카드 첫 작성 — 세 가지 유형

아래 세 유형을 각각 1~3개씩 적습니다. 다 적으면 AI 피드백을 받아 구체적인 계획으로 다듬고(맨 아래 안내), 그룹에 공유한 뒤 문제·전략 은행에 카드로 등록합니다.

### ① 해결하고 싶은 문제 — AI로 시도해볼 일
(반복적으로 시간을 잡아먹는 일, 미뤄둔 문제)

- 자료 및 데이터 정리: 입시 정책 및 결과, 대학별 모집요강 전형별 자료 등
- 반복적인 행정업무 자동화
- 평가 서류 검토의 1차 반복 작업 시스템화

### ② 늘리고 싶은 역량 — 아직 서투른 판단·기술
(잘하는 것과 막히는 것을 구체적인 장면으로)

- 프롬프트/지침 작성 역량
- Agent 워크플로우 설계 역량
- Agent 결과에 대한 감독 및 검증 역량

### ③ 깊이 알고 싶은 것 — 내 분야에서 설명 못 하는 것, 늘 궁금했던 것

- 교육 데이터 특화 Agent 활용 사례 — 입학사정관 업무나 유사 정성평가 영역에서 AI Agent를 실제 도입한 국내외 사례가 있는지
- AI Agent 판단 근거의 신뢰 조건 — Agent가 정성적 서류를 평가·요약할 때 어디까지 신뢰할 수 있고, 어떤 검증 장치가 필요한지
- Agent 도입 시 개인정보·윤리 이슈 — 학생 개인정보가 담긴 서류를 Agent가 다룰 때 지켜야 할 원칙과 제약은 무엇인지

---

### ##### AI 피드백 반영 버전 #####

#### ① 해결하고 싶은 문제

1. **자료 및 데이터 정리**
   - 우선 3~4개 대학, 학생부종합 전형으로 범위 한정해 관련 파일 폴더로 모으기
   - 정한 범위 안에서 전형 유형별 비교표 작성 요청
   - 정리된 내용을 원문과 대조해 오류 확인
   - 재사용 가능한 프롬프트 템플릿으로 저장

2. **반복적인 행정업무 자동화**
   - 최근 반복됐던 문의·공문 유형 목록화
   - 유형별 응대 초안 프롬프트 작성
   - 최근 실제 받았던 문의 1~2건을 다시 넣어 재현 테스트 (신규 문의 발생 여부에 의존하지 않도록)

3. **평가 서류 검토의 1차 반복 작업 시스템화**
   - 기준으로 삼을 문서 먼저 확정 (우리 기관 자체 평가 매뉴얼 또는 교육부 정성평가 가이드라인)
   - 1개 전형으로 한정해 지원자별 핵심정보 추출 항목 정의
   - 샘플 서류 1~2건으로 요약표 초안 작성 요청
   - 검증 기준(어떤 항목이 맞으면 통과인지) 한 줄로 정하고 결과 확인

#### ② 늘리고 싶은 역량

1. **프롬프트/지침 작성 역량**
   - 결과가 아쉬웠던 기존 프롬프트 1~2개 선정
   - 참고자료는 Anthropic 프롬프트 엔지니어링 가이드로 지정해 짧게 훑기 (30분 내외)
   - 나머지 시간은 실제 프롬프트 재작성·템플릿 제작에 집중 배분

2. **Agent 워크플로우 설계 역량**
   - 반복 작업 하나를 골라 단계별로 쪼개보기
   - 사용 도구를 먼저 지정 (예: Claude Code) 후 워크플로우 구성
   - 완성 후 막혔던 지점 기록

3. **Agent 결과에 대한 감독 및 검증 역량**
   - 결과물 검증용 체크리스트 초안 작성 (사실확인·누락·톤 등)
   - ①-3에서 만든 요약표 결과물을 그대로 재사용해 체크리스트 적용
   - 검증 중 발견한 오류 유형 기록

#### ③ 깊이 알고 싶은 것

1. **교육 데이터 특화 Agent 활용 사례**
   - 관련 키워드로 국내외 사례 검색
   - 발견한 사례 2~3건 요약 정리
   - 우리 업무에 적용 가능/불가능한 부분 구분

2. **AI Agent 판단 근거의 신뢰 조건**
   - "정성평가·문서요약" 맥락으로 한정해 신뢰성·설명가능성 기본 개념 정리
   - 업무에서 어디까지 믿을 수 있을지 기준 초안 작성
   - 동료와의 논의는 별도 일정으로 분리하고, 오늘은 "논의할 질문 목록"만 작성

3. **Agent 도입 시 개인정보·윤리적 이슈**
   - 학생 개인정보 관련 기존 규정·지침 확인: 범위를 우리 기관의 학생부 취급 규정으로 한정해 확인
   - Agent 사용 원칙 초안 작성 (익명화, 접근권한 등)
   - 기관 정책 담당자와 확인 필요한 부분 목록화

다 적었다면
① AI에게 이 목록을 보여주고 피드백을 받으세요: 애매한 항목은 어디인지, 빠진 것은 없는지, 각 항목이 하루(작업 약 3시간)에 시도할 만한 크기인지.
② 피드백을 참고해 항목마다 첫 데이에 할 구체적인 계획을 내 판단으로 확정하세요 — AI의 제안을 그대로 받아쓰지 말고, 내 상황에 맞게 고칩니다.
③ 계획을 그룹에 공유하고, 확정된 항목을 문제·전략 은행에 카드로 등록하세요. 이후 매 데이의 작업 결과가 이 카드들 위에 쌓이고, 그 기록을 다시 그룹과 공유합니다.
