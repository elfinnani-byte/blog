# 💻 [비전공자 바이브 코딩] 3. AI 에이전트와 PRD로 내 컴퓨터에 Todo 앱 DB 연동해보기

안녕하세요! **'바이브 코딩으로 웹 만들기'** 시리즈의 마지막 3편입니다.

이번 편에서는 앞서 배운 데이터베이스 개념(SQL/NoSQL, CRUD, 보안)을 바탕으로, **AI 에이전트(Cursor, Claude Code 등)에게 PRD를 제공하여 로컬 DB(SQLite)와 클라우드 DB(Supabase)를 연동한 Todo 앱을 만드는 실습 과정**을 정리합니다.

---

## 1부: PRD(제품 요구사항 문서) 작성과 DB 구조 설계 받기

개발을 시작하기 전, 에이전트가 어떤 앱을 만들어야 할지 명확한 지침서인 **PRD.md**를 먼저 작성했습니다.

* **목표**: 내 컴퓨터에서 실행되며, 데이터를 안전하게 유지하는 Todo 앱 제작
* **기본 기능 (CRUD)**: 할 일 추가(C), 목록 조회(R), 완료 표시(U), 삭제(D)
* **커스텀 기능**: 마감일 설정 및 카테고리(태그) 분류 (다대다 관계)
* **규칙**: 보안을 위해 `.env` 사용, `.gitignore` 설정 필수

### 💡 AI 에이전트와의 첫 대화 (DB 구조도 요청)
> *"위 PRD로 만들 Todo 앱의 DB 구조(어떤 표·열·관계)를 먼저 설계해서 구조도로 그려 보여줘. 로컬 무료 DB(SQLite)를 쓸 거야."*

에이전트가 `todos` 테이블과 `tags` 테이블, 그리고 둘을 이어주는 연결 테이블(`todo_tags`)로 구성된 **1:N 및 N:M 관계 구조도**를 제안했고, 이를 확인한 후 구현을 진행했습니다.

---

## 2부: 길 A - 내 컴퓨터(로컬)에서 돌리기 (SQLite)

인터넷 없이 내 PC에서 완전히 작동하도록 **SQLite**를 활용해 구현을 지시했습니다.

### 🤖 실행 지시 프롬프트
> *"이 PRD와 DB 구조로 Todo 앱을 만들어 줘. 데이터는 로컬 SQLite 파일 하나(`todo.db`)에 저장하고 브라우저(`localhost`)로 열리게 해 줘. `.env`와 `todo.db`는 `.gitignore`에 넣어줘. 먼저 전체 계획을 보여주고 각 단계 끝에 확인 지점을 알려줘."*

### 🛠️ 개발 과정 및 점검 포인트
1. **에이전트의 계획 검토**: PRD 범위에서 벗어나지 않는지 확인 후 승인
2. **로컬 데이터 저장 확인**: 앱을 껐다 켜거나 브라우저를 새로고침해도 할 일 목록이 잘 유지되는지 테스트
3. **자주 발생하는 트러블슈팅**:
   * `better-sqlite3` 빌드 에러 시 에러 메시지를 그대로 에이전트에 전달하여 C++ 빌드 도구 설치 해결
   * `todo.db` 파일이 깃허브에 올라가지 않도록 `.gitignore` 세팅 재확인

---

## 3부: 길 B - 클라우드 DB(Supabase) 연동하여 배포 준비하기

로컬 완성이 끝난 뒤, 남들에게 링크로 보여주거나 24시간 실행되도록 **Supabase(클라우드 Postgres DB)**로 전환을 진행했습니다.

### 🤖 클라우드 전환 프롬프트
> *"같은 Todo 앱을 로컬 SQLite 대신 Supabase(무료 클라우드 Postgres)에 저장하도록 바꿔 줘. 접속 키는 `.env`로 빼고, RLS(행 수준 보안)를 켜서 허용된 접근만 되게 해 줘."*

* **Supabase 대시보드 확인**: Table Editor에서 실제 데이터 행이 쌓이는 것을 확인
* **RLS(Row-Level Security) 설정**: 공개 API 키가 노출되더라도 권한 없는 접근을 막도록 보안 규칙 적용
